Example format:

FEATURE: ADD PRODUCT TO CART

Trigger:
User taps the Add button.

Preconditions:
User session state is known.
Product is active.
Product is available.

System Behaviour:

1. Validate product availability.
2. Check current stock.
3. Determine current cart.
4. Add the product to the cart.
5. Set quantity to 1 if the item does not exist.
6. If the product already exists, follow the defined quantity update behaviour.
7. Persist the cart state.
8. Refresh the cart query.
9. Update the product card UI.
10. Update the cart badge.

Loading Behaviour:
Disable repeated Add actions while the mutation is pending.

Success Behaviour:
Replace Add button with quantity controls.

Failure Behaviour:
Restore the previous UI state.
Display a human-readable error.

Edge Cases:
Product becomes inactive.
Product becomes out of stock.
Network disconnects.
User taps Add repeatedly.
Database operation fails.
